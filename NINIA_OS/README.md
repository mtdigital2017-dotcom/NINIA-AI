# NINIA_OS — memoria operativa del proyecto

Esta carpeta es la fuente de continuidad técnica de NINIA-AI.

## Regla de trabajo

1. Al iniciar una sesión, leer `DT_MEMORY.md` y `CURRENT_SPRINT.md`.
2. Antes de cambiar arquitectura, registrar la decisión en `DECISIONS_LOG.md`.
3. Antes de un `git push`, ejecutar el checklist de `CHECKLISTS.md`.
4. Al cerrar un avance, actualizar `CHANGELOG.md`, `PROJECT_STATE.md` y `LEARNING_LOG.md`.
5. GitHub es la fuente oficial versionada. Google Drive conserva respaldos automáticos verificados.

## Perfil del responsable del proyecto

El responsable está aprendiendo desarrollo de software. Toda instrucción debe explicar:

- qué se hará;
- por qué;
- qué archivo cambia;
- qué riesgo existe;
- cómo verificar el resultado.
